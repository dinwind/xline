# Axline 用户使用手册

> **适用版本**：VS Code 扩展 `0.2.7`（扩展 ID：`axline.axline`）  
> **适用对象**：通过企业私有 VSIX 分发、使用 AxGate 账号登录的内部开发者  
> **最后更新**：2026-07-15

---

## 1. Axline 是什么

Axline 是集成在 VS Code 中的 **自主编码 Agent**。它可以：

- 阅读并理解你的项目结构
- 在获得你批准后修改文件、运行终端命令
- 通过 MCP 连接数据库、API 等外部工具
- 使用企业 AxGate 网关提供的 AI 模型（无需自行配置云厂商 API Key）

Axline 基于开源项目 [Cline](https://github.com/cline/cline) 定制，面向需要 **企业身份认证** 与 **私有扩展更新** 的团队。

---

## 2. 系统要求

| 项目 | 要求 |
|------|------|
| 编辑器 | Visual Studio Code **1.84** 或更高版本 |
| 操作系统 | Windows / macOS / Linux |
| 网络 | 可访问企业 AxGate 与 AuthNexus 服务 |
| 账号 | 管理员已在 AuthNexus 为你开通 `axline` 应用访问权限 |

## 3. 安装

### 3.1 从 VSIX 安装（推荐）

企业环境通常通过 AuthNexus 私有更新或运维分发 VSIX，而非公共扩展市场。

1. 获取最新 `axline-enterprise.vsix`（或 `axline.vsix`）
2. 打开 VS Code → **扩展**（Extensions）
3. 点击扩展视图右上角 `...` → **从 VSIX 安装…**（Install from VSIX…）
4. 选择下载的 VSIX 文件
5. 安装完成后 **重新加载窗口**（Reload Window）
6. 在左侧 **活动栏**（Activity Bar）找到 **Axline** 图标并点击打开

### 3.2 命令行安装（可选）

```powershell
# Windows PowerShell — 抑制 VS Code CLI 的无害弃用警告
$env:NODE_OPTIONS="--disable-warning=DEP0169"
code --install-extension path\to\axline-enterprise.vsix --force
```

### 3.3 安装后检查

- 活动栏出现 Axline 图标
- 首次打开侧边栏应显示 **Sign in to Axline** 登录页（不再出现 Get Started 引导向导）
- 扩展版本可在 **扩展详情页** 或命令面板中查看

## 4. 首次启动与登录

### 4.1 登录流程

1. 点击活动栏 **Axline** 图标，打开侧边栏
2. 在登录页输入 **邮箱或用户名** 与 **密码**
3. 点击 **Sign in**
4. 登录成功后自动进入主聊天界面

登录使用企业 **AuthNexus** 账号，经 **AxGate** 网关鉴权。你无需、也不应自行填写 OpenAI / Anthropic 等 API Key。

### 4.2 登录前提

- 管理员已将你添加为 AuthNexus 中 `axline` 应用的成员
- 本机可访问 AxGate 服务地址（企业 VSIX 通常已内置；见第 8 节配置说明）
- 若提示无法连接或配置缺失，请联系平台管理员，或检查 `~/.axline/endpoints.json`

### 4.3 会话保持

- VS Code **保持打开** 时：令牌临近过期会自动刷新，一般无感续期
- VS Code **关闭超过约 1 小时**（令牌生命周期）：需重新输入用户名和密码登录

### 4.4 常见登录错误

| 现象 | 可能原因 | 处理建议 |
|------|----------|----------|
| 401 / 凭据无效 | 用户名或密码错误 | 核对账号后重试 |
| 403 / 权限不足 | 未开通 `axline` 应用 | 联系管理员添加应用成员 |
| 无法连接服务器 | 网络或 `axgateBaseUrl` 配置错误 | 检查网络与 `~/.axline/endpoints.json` |
| 429 / 请求过多 | 登录被限流 | 稍后再试，勿频繁重复提交 |

## 5. 界面概览

打开 Axline 侧边栏后，顶部工具栏提供以下入口：

| 按钮 | 功能 |
|------|------|
| **+**（New Task） | 开始新任务 / 新对话 |
| **扳手**（Customize） | 自定义：规则、Skills、Hooks 等 |
| **历史**（History） | 查看过往任务记录 |
| **账号**（Account） | 用户信息、用量、Provider 概览 |
| **设置**（Settings） | 模型、功能与行为设置 |

主区域为 **聊天输入框** 与 **对话历史**。Agent 的每一步操作（读文件、改代码、跑命令）都会在对话中展示，并等待你批准或拒绝。

## 6. 基本使用

### 6.1 发起任务

在输入框中用自然语言描述目标，例如：

- 「为这个 API 添加单元测试」
- 「解释 `src/config.ts` 里的配置加载逻辑」
- 「修复终端里显示的 TypeScript 编译错误」

按 **Enter** 发送（Shift+Enter 可换行，具体以界面为准）。

**提示**：尽量说明 **目标** 而非零散指令，例如「实现用户登录接口并补充测试」比「写个函数」效果更好。

### 6.2 人工审批（Human-in-the-Loop）

Axline 在修改文件或执行终端命令前会请求你的确认：

1. 查看 Agent 提出的 **计划** 或 **diff 预览**
2. 点击 **批准**（Approve）继续，或 **拒绝**（Reject）中止该步骤
3. 可在设置中配置 **自动批准**（Auto-approve）以加快流程——仅在信任的工作区内使用

所有文件变更以 **diff** 形式展示，并支持 **检查点（Checkpoint）** 回滚，便于撤销 Agent 的修改。

### 6.3 附加上下文

- **@ 提及**：在输入框输入 `@` 可引用文件、文件夹等上下文
- **选中代码**：在编辑器中选中代码后，使用快捷操作（见第 7 节）添加到对话
- **终端输出**：在终端中选择内容，通过右键菜单 **Add to Axline** 附带错误日志或命令输出

### 6.4 任务历史

点击 **History** 可查看、恢复或参考以往任务。适合在长周期开发中接续未完成的工作。

### 6.5 企业账号模式说明

在 AxGate 企业登录模式下，扩展以 **统一网关模型** 为主路径；界面侧重于账号权限内的可用模型，而非自行配置多家 API Provider 的入门向导。

## 7. 快捷操作与命令

### 7.1 键盘快捷键

| 操作 | Windows / Linux | macOS |
|------|-----------------|-------|
| 聚焦聊天输入（无选中时） | `Ctrl+'` | `Cmd+'` |
| 将选中代码加入对话 | `Ctrl+'`（有选中时） | `Cmd+'`（有选中时） |

### 7.2 编辑器右键菜单

在代码编辑器中右键可使用：

- **Explain with Axline** — 解释选中代码
- **Improve with Axline** — 改进选中代码
- **Add to Axline** — 将选中内容加入当前对话

### 7.3 命令面板

按 `Ctrl+Shift+P`（macOS：`Cmd+Shift+P`），输入 `Axline` 可找到：

| 命令 | 说明 |
|------|------|
| **Axline: Check for Updates** | 手动检查私有更新 |
| **Axline: Open Walkthrough** | 打开功能引导（可选阅读） |
| **Axline: Jump to Chat Input** | 跳转到聊天输入框 |
| **Axline: Generate Commit Message with Axline** | 在 Git 提交界面生成提交说明 |
| **Axline: New Task** | 新建任务 |

### 7.4 Jupyter 笔记本（如已安装 Jupyter 扩展）

在 Notebook 单元格上可使用生成、解释、改进等 Axline 命令。

## 8. 配置与数据目录

Axline 将用户数据存放在用户主目录下的 **`~/.axline`**（Windows 示例：`C:\Users\<你>\.axline`）。

### 8.1 目录结构（常用）

```
~/.axline/
├── endpoints.json          # 公开服务端点（AxGate、AuthNexus 等）
├── secrets.json            # 敏感凭据（更新安装码等，勿分享）
├── data/                   # 状态、设置、Skills、MCP 配置等
│   └── settings/
│       └── axline_mcp_settings.json
└── update/                 # 私有更新相关缓存
```

### 8.2 endpoints.json

首次启动时，扩展会 **自动创建** `endpoints.json`（从企业 VSIX 内置配置、遗留 `~/.cline` 或默认值）。典型内容：

```json
{
  "axgateBaseUrl": "https://auth.mtsilicon.com:6343",
  "authAppId": "app_uu1Sn7yC",
  "authNexusBaseUrl": "https://auth.mtsilicon.com",
  "updateAppId": "app_uu1Sn7yC"
}
```

- 一般用户 **无需修改**，除非管理员另行通知切换环境
- 该文件 **不含密码**，可交由运维统一分发模板

### 8.3 secrets.json（私有更新）

若需启用 AuthNexus 自动更新检查，由运维或本机一次性配置：

```json
{
  "updateEnrollmentCode": "<Permanent 安装码>"
}
```

**注意**：`secrets.json` 含敏感信息，请勿提交到 Git 或发送给他人。

### 8.4 从旧版 Cline 迁移

- 若你曾使用 `~/.cline`，扩展会在首次运行时尝试将 `~/.cline/data` **复制** 到 `~/.axline/data`
- 旧的 `~/.cline/endpoints.json` 仍可能被读取，但建议迁移到 `~/.axline`
- MCP 设置文件会从 `cline_mcp_settings.json` **自动重命名** 为 `axline_mcp_settings.json`

### 8.5 VS Code 设置

在 **设置** 中搜索 `Axline`：

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| `axline.update.autoCheck` | `true` | 启动后及周期检查私有更新 |
| `axline.update.checkIntervalHours` | `24` | 自动检查间隔（小时） |

环境变量（高级用户 / 运维）：

| 变量 | 说明 |
|------|------|
| `AXLINE_DIR` | 覆盖 `~/.axline` 主目录 |
| `AXLINE_NO_UPDATE_CHECK=1` | 禁用更新检查 |
| `AXLINE_UPDATE_ENROLLMENT_CODE` | 覆盖 secrets 中的安装码 |

## 9. 账号与用量

登录后点击 **Account** 可查看：

- 当前登录身份（用户名 / 邮箱等展示信息）
- 经 AxGate 分配的 **Provider** 与可用模型
- **Token / 配额用量**（以企业网关配置为准）

**说明**：

- 你看到的模型与额度由企业管理员在 AxGate / AuthNexus 侧配置，客户端无需填写 API Key
- 若某模型不可用或提示权限不足（403），请联系管理员调整 `axline` 应用角色或配额
- 点击 **登出** 会清除本地凭证，需重新登录后方可继续对话

## 10. 私有自动更新

企业版 Axline 通过 **AuthNexus 软件发布 API** 检查并安装新版本 VSIX。

### 10.1 自动更新

满足以下条件时，扩展会在启动时及每隔一定时间自动检查：

- 已配置 `authNexusBaseUrl`、`updateAppId` 与 `updateEnrollmentCode`
- `axline.update.autoCheck` 为 `true`
- 非开发调试模式（F5 扩展开发）

发现新版本时会提示下载安装，安装后通常需 **重新加载窗口**。

### 10.2 手动检查

命令面板执行 **Axline: Check for Updates**。

### 10.3 禁用更新检查

- 设置 `axline.update.autoCheck` 为 `false`，或
- 设置环境变量 `AXLINE_NO_UPDATE_CHECK=1`

## 11. MCP 扩展能力

**MCP**（Model Context Protocol）允许 Axline 连接外部工具，例如 Jira、数据库、内部 API 等。

### 11.1 管理 MCP 服务器

1. 在 Axline 侧边栏点击 **MCP Servers**（服务器图标），或从 Customize 入口进入
2. 添加、启用、禁用或删除 MCP 服务器
3. 配置保存在 `~/.axline/data/settings/axline_mcp_settings.json`

### 11.2 使用建议

- 仅添加你信任且经企业安全评审的服务器
- OAuth 类 MCP 首次连接可能需在浏览器中完成授权
- 在对话中 Agent 会自动调用已启用且授权成功的 MCP 工具

## 12. 项目规则、Skills 与 Hooks

### 12.1 项目规则（.clinerules）

在仓库根目录或 `.clinerules/` 下放置规则文件，可约束 Agent 在本项目中的行为，例如：

- 编码规范与目录约定
- 测试与提交要求
- 禁止修改的路径

Axline 打开该工作区时会 **自动加载** 这些规则。

### 12.2 Skills

Skills 是可按需加载的能力说明（如发布流程、调试 SOP）。常见位置：

- 用户级：`~/.axline/data` 下相关 skills 目录
- 项目级：仓库内 `.cline/skills` 或 `.agent/skills`（视团队配置而定）

通过 **Customize** 入口可浏览与管理。

### 12.3 Hooks

Hooks 可在 Agent 执行特定阶段（如提交前、工具调用前）运行脚本，用于审计、策略检查等。由团队统一维护，普通用户通常只需知晓其存在并在报错时联系维护者。

## 13. 典型工作流示例

### 13.1 修复编译错误

1. 在终端运行构建，复制报错信息
2. 选中报错 → **Add to Axline**，或粘贴到输入框
3. 发送：「请根据以上错误修复项目中的类型问题」
4. 逐步批准 Agent 的文件修改，直至构建通过

### 13.2 理解陌生模块

1. `@` 引用目标文件或目录
2. 提问：「说明该模块的职责、主要入口和与上下游的依赖关系」
3. 根据回答继续追问或要求画调用关系（Agent 会以文字说明）

### 13.3 生成功能代码

1. 用一句话描述业务目标与约束（接口格式、测试要求等）
2. 批准 Agent 的实现计划
3. 审查 diff，运行项目测试
4. 使用 **Generate Commit Message with Axline** 生成提交说明

## 14. 常见问题（FAQ）

**Q：安装后没有 Axline 图标？**  
A：确认 VSIX 安装成功并已 Reload；在扩展列表中启用 Axline；必要时重启 VS Code。

**Q：一直停在登录页，无法聊天？**  
A：必须完成 AxGate 登录；检查账号是否已开通 `axline` 应用及网络是否可达。

**Q：提示 AxGate 未配置？**  
A：检查 `~/.axline/endpoints.json` 是否存在且 `axgateBaseUrl` 正确；企业 VSIX 通常已内置，可尝试重新安装最新 enterprise 包。

**Q：更新检查失败？**  
A：确认 `secrets.json` 中 `updateEnrollmentCode` 有效，且能访问 `authNexusBaseUrl`；可用手动命令 **Check for Updates** 查看具体错误。

**Q：Agent 改了不该改的文件？**  
A：使用 Checkpoint 回滚；在 `.clinerules` 中明确禁止路径；关闭 auto-approve，逐步审批。

**Q：能否使用自己的 OpenAI Key？**  
A：企业部署以 AxGate 统一供模为主；是否允许 BYOK 取决于管理员策略，默认路径为账号登录而非 API Key 向导。

**Q：与公共版 Cline 文档的差异？**  
A：上游 [Cline 文档](https://docs.cline.bot) 仍可作为功能参考，但安装、登录、更新、配置目录请以本手册及企业运维说明为准。
